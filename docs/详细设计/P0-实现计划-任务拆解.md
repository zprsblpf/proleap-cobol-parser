## Task 5：悬空 `END-IF` 配平（P0 最难点 · 重点验证项）

> 现象：源里 `IF` 行被 `!!!!!!` 停用、其 body `GO TO` + `END-IF` 仍激活 → 停用 IF 后留孤儿 `END-IF` → ProLeap 严格 AST 致命中断。
> 方案优先级（父设计 §2.4）：**首选预处理层**——停用某 `IF` 时，用 IF/END-IF 计数栈配平、一并作废其作用域内对应 `END-IF`；文法/错误恢复容错为次选，仅当预处理不可行再考虑。
