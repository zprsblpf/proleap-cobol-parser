## 2. 改动落点（ProLeap 三层）

> 精确类名/行号由实现会话现场考古定位（避免高 context 下臆测）；下表给定位线索与 spike 已知锚点。

### 2.1 预处理 · 行清理层（FIXED 列处理）
spike 已知锚点：`CobolPreprocessorImpl` / `CobolDocumentParserImpl` 一带的按格式清行逻辑。
- **DBCS 双宽列对齐**（ProLeap 真短板，引擎内修最干净）：FIXED 格式列截断（7–72 / 73–80 区划分）
  按**显示宽度**计算，CJK 全角字符占 2 列，而非按字符数。根治中文行 `<变更标记>` 被算进代码区的错位。
- **`!!!!!!` 停用行**：cols 1–6 全为 `!` → 整行作废（等同注释/空行），不进文法。
- **行尾变更标记 `<XXXX>`**：DBCS 对齐后多落在 72 列外被自然截断；加显式兜底去除（保守正则，不误伤 `<` 比较符）。

### 2.2 预处理 · COPY 解析层（缺簿容忍）
spike 已知锚点：`CobolDocumentParserListenerImpl` 第 341 行 `throw CobolPreprocessorException("Could not find copy book ...")`。
- 新增参数 `ignoreMissingCopyBooks`（或等价开关）：缺簿时**跳过/插空串**、不抛异常 → **免造 144 桩拷贝簿**。
- 缺簿清单仍记录输出（供排查），但不阻断解析。

### 2.3 文法层（方言写法）
锚点：`src/main/antlr4` 下 `CobolParser.g4` 的 `goToStatement` 规则。
- **`GO` 省略 `TO`**：让 `TO` 可选（`GO TO X` 与 `GO X` 均合法），改后重生成 parser（antlr4 maven 插件已在构建链）。

### 2.4 P0 最难点 · 悬空 END-IF（重点验证项）
现象：源里「`IF` 行被 `!!!!!!` 停用、其 body `GO TO` + `END-IF` 仍激活」→ 停用 IF 后留下孤儿 `END-IF`，ProLeap 严格 AST 致命中断。
方案优先级：
1. **预处理层解决（首选）**：停用某 `IF` 时，配套作废其作用域内对应的 `END-IF`（用 IF/END-IF 计数栈配平，或在停用块内一并清理）。
2. 文法/错误恢复容错（次选，仅当预处理不可行时再考虑）。
spec 要求：实现时**专门验证此类构造**，并在覆盖率报告中单列「悬空 END-IF 处理结果」。
